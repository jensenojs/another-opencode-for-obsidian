# Another OpenCode Plugin for Obsidian v0.3.0

This release focuses on making OpenCode practical to use inside Obsidian. The
plugin still uses OpenCode's own Web UI as the conversation surface, but it now
handles more of the Obsidian-specific behavior around that UI.

## Highlights

- Added an Obsidian/OpenCode keyboard bridge.
  - Obsidian hotkeys and OpenCode keybinds are normalized into one shortcut
    index.
  - Shortcut conflicts are visible in the plugin settings panel.
  - Each conflict can be assigned to Obsidian or OpenCode.
- Added deep view for OpenCode.
  - `Mod+Shift+L` opens OpenCode in the main editor area.
  - Triggering the command again returns to the previous editor/content leaf.
- Improved OpenCode Web UI integration inside Obsidian.
  - Vault paths, wikilinks, headings, blocks, footnotes, diff rows, and markdown
    paths can navigate back to existing Obsidian notes.
  - Missing navigation targets do not create new vault files.
  - Obsidian-derived appearance remains available for the embedded Web UI.
- Improved context visibility and diagnostics.
  - Workspace and selection context retain source metadata, provenance, restore
    state, and safe navigation targets.
  - Status bar context state and native OpenCode context cards are kept
    consistent where the Web UI exposes the required bridge.
  - Runtime diagnostics include keyboard policy, context projection state,
    bridge state, theme state, server state, and XDG log/status paths.
- Refreshed public documentation.
  - Added a Chinese README.
  - Rewrote the README around the current product: OpenCode usable inside
    Obsidian.
  - Moved GraphRAG wording into optional future work rather than the main
    product path.
  - Added `versions.json` for Obsidian version compatibility metadata.

## Installation

BRAT users can install this release from:

```text
jensenojs/another-opencode-for-obsidian
```

The release includes the BRAT-required assets:

- `manifest.json`
- `main.js`
- `styles.css`

## Notes

- This is still beta software.
- GraphRAG remains optional research work. The current product does not depend
  on it.
- In path mode, the OpenCode server inherits the Obsidian desktop process
  environment. If `opencode serve` works in Terminal but fails from Obsidian,
  compare `processEnvironment` and `lastSpawnEnvironment` in diagnostics before
  changing startup settings.
